# A股多因子项目专业时间序列研究执行计划

## 一、目标、范围与当前起点

### 1. 总目标

建立一套严格样本外、可复现、可审计的专业时间序列研究体系，回答四个核心问题：

1. 因子有效性是否存在可预测的时间漂移？
2. 市场状态是否能够改善因子配置和回撤控制？
3. 条件波动预测是否能够改善组合风险收益比？
4. 数据、模型、回测和审计证据是否能够通过一个只读研究看板被持续追踪和复核？

最终策略结构为：

```text
截面因子分数
× 动态因子权重
× 市场状态调整
× 波动率目标缩放
→ 结构化研究证据
→ 只读前端看板
```

### 2. 第一阶段边界

第一阶段不将以下模型投入主生产研究：

- PatchTST、TCN 等深度网络；
- 全股票 VAR/VECM；
- 全股票 DCC-GARCH；
- 高频日内模型；
- 大规模自动超参数搜索；
- 直接预测股票价格水平；
- 逐股票独立训练深度模型。

### 3. 当前工程起点

截至 2026-07-19，当前项目已经具备：

- 时间序列配置契约；
- 月度因子 IC 和因子收益构造；
- HMM、Kalman、GJR-GARCH 和 DCC 第一版；
- DM、SPA、DSR 和 PBO 审计；
- `fit/update/forecast` 统一接口；
- 真实模式历史准入和禁止降级机制；
- 共同投资区间绩效比较；
- 冻结的 2015/2018/2024 真实研究协议与试验登记路径；
- 数据源登记、许可/PIT 审查和 `data_manifest.json` v2；
- 不可覆盖批次、断点续取和正式股票代码文件入口；
- 财务修订链保留及按信号日选择当时最新可用版本；
- PIT 时点、财务修订、幸存者偏差、逐日覆盖率、基准对齐和总门禁六项审计；
- 108 项通过的自动化测试、sample smoke 和真实模式负向 smoke；
- **（2026-07-19 新增）** AkShare 开源核验与全量抓取批次 `data/raw/real-20260719`：交易日历、中证 500 基准（新浪备用源）、5,533 条证券主表（含 333 只退市股）、1,663,139 行日行情（833 只股票池，500/500 当前成分覆盖，新浪+腾讯双源，新浪复权因子快照替代占位因子）；
- **（2026-07-19 新增）** 12 份数据源审查证据文件、缺口数据提供规范与 7 张缺失表模板（`reports/data_sources/`）；
- **（2026-07-19 新增，2026-07-20 纠错）** 4 张表的审查材料已备齐但登记状态仍为 `pending_user_review`；新浪 `qfq-factor` 按除数语义转换后，纠错暂存与部分标准导入分别保存为 `data/staging/real-20260719-r1` 和 `data/standard/real-stage-20260719-r1`，基准对齐审计 2,803/2,803 交易日通过。旧的无 `-r1` 暂存/标准产物已被取代，不得用于收益计算或签署。

当前状态分为两个层次：

- **工程验收：已完成。** 协议、导入、manifest、六项审计、真实运行硬阻断和回归测试已经落地。
- **数据验收：阻断。** 7 张许可表（历史成分、估值、财务修订、行业、停复牌、ST、涨跌停）仍待用户提供，登记表与签署页已备妥待审；真实门禁继续保持 `blocked_by_missing_pit_tables`。

当前尚未完成：

- 2015 年至今的真实 PIT 数据接入；
- 至少 36 个月可用的动态方案 OOS 结果；
- 模型稳定性和超参数研究；
- 三方案真实消融比较；
- 最终模型晋级结论。
- 前端看板目录、数据契约和验收基线。

Kimi CLI 不作为研究数据来源，也不读取私有仓库。当前没有可用的 iFinD 连接器、SDK 凭据或经审查授权，因此 iFinD 从当前关键路径移除；如未来用户另行提供合法授权，再作为可选补充来源重新立项和审查。

## 三、阶段 0：冻结研究协议与基线

### 目标

在接入真实数据前固定研究对象、时间区间、模型清单和晋级标准，避免看到结果后修改研究规则。

### 执行任务

- [x] 在研究协议中固定数据开始日、评价开始日和最终留出期。
- [x] 固定预测频率为月度、预测步长为 1 个月。
- [x] 固定因子择时、状态识别和波动模型候选清单。
- [x] 固定 DM、SPA、DSR、PBO 的使用规则。
- [x] 建立试验登记表，每个模型或参数组合分配唯一 `experiment_id`。
- [x] 冻结合成数据基线，运行目录存在且非空时禁止覆盖。

建议协议配置：

```yaml
research:
  data_start: "2015-01-01"
  evaluation_start: "2018-01-01"
  final_holdout_start: "2024-01-01"
  frequency: monthly
  forecast_horizon_months: 1
  minimum_oos_months: 36

model_registry:
  factor_timing:
    - static_equal
    - rolling_ic_12m
    - rolling_ic_24m
    - ewma_ic
    - kalman_local_level
  regime:
    - no_regime
    - rule_based
    - hmm_2_state
    - hmm_3_state
  volatility:
    - no_scaling
    - historical_vol
    - ewma
    - gjr_garch

selection:
  primary_metric: net_information_ratio
  prediction_tests:
    - dm
    - spa
  overfit_tests:
    - dsr
    - pbo
```

### 预注册研究假设

- **H1：**Kalman 动态权重扣费后优于静态等权和滚动 IC。
- **H2：**实时 HMM 状态调整可以降低回撤，而不显著损害长期收益。
- **H3：**GJR-GARCH 波动目标可以降低实际波动偏离和尾部损失。

### 验收标准

- [x] 协议包含全部模型、窗口和评价门槛。
- [x] 后续新增模型必须登记试验编号。
- [x] 当前样例结果明确标记为 `synthetic_fallback`。
- [x] 样例结果不得用于真实市场有效性结论。

## 四、阶段 1：真实 PIT 数据接入与质量审计

### 1. 数据范围

#### 行情数据

- `trade_date`、`ts_code`；
- 开、高、低、收和前收盘价；
- 成交量、成交额；
- 复权因子；
- 停牌标记；

- 上市日期和退市日期；
- ST 状态有效区间；
- 行业分类有效区间。

- 公告日期；

### 1-A. 当前数据来源分层

数据来源优先级固定为：

```text
交易所、指数公司、上市公司公告等官方来源
  > 可验证的合法开源数据
  > 用户提供且完成来源、许可和 PIT 审查的本地标准表
```

当前固定数据路线：

- AkShare 只正式承接交易日历、日行情和中证 500 指数行情；
- 当前股票列表只可用于侦察，不满足历史股票池门禁；
- 历史成分、证券主表、退市/ST/停复牌、涨跌停、行业、估值和财务修订由用户提供本地合法 PIT 标准表；

## 七、阶段 4：Kalman 动态因子权重

> 这是第一优先研究模块。

### 1. 模型定义

对每个因子的月度 IC 建立局部水平模型：

$$
IC_{i,t}=\mu_{i,t}+\epsilon_{i,t}
$$

$$
\mu_{i,t}=\mu_{i,t-1}+\eta_{i,t}
$$

其中：

- $\mu_{i,t}$：潜在因子有效性；
- $\epsilon_{i,t}$：观测噪声；
- $\eta_{i,t}$：因子有效性漂移。

### 2. 权重构造

第一版权重分数：

$$
s_{i,t}=\frac{\hat{\mu}_{i,t}}{\hat{\sigma}_{i,t}^{2}+\lambda}
$$

依次执行：

1. 因子方向调整；
2. FDR 过滤；
3. 覆盖率过滤；
4. 单因子权重上限；
5. 权重归一化；
6. 换手惩罚。

换手平滑：

$$
w_t=(1-\rho)w_t^{raw}+\rho w_{t-1}
$$

默认 $\rho=0.2$。

### 3. 预注册参数网格

```yaml
process_variance:
  - 0.0001
  - 0.001
  - 0.01

observation_variance:
  - 0.005
  - 0.01
  - 0.02

turnover_penalty:
  - 0.0
  - 0.2
  - 0.5
```

27 组参数必须全部登记为试验次数。

### 4. 对照组

- 静态等权；
- 方向锁定后的等权；
- 12 个月滚动 IC；
- 24 个月滚动 IC；
- EWMA IC；
- Kalman IC。

### 5. 输出

```text
dynamic_factor_weights.csv
factor_weight_turnover.csv
factor_weight_stability.csv
factor_ic_forecasts.csv
factor_timing_comparison.csv
```

### 晋级标准

- [ ] 至少 36 个月非重叠 OOS。
- [ ] 扣费后优于静态等权。
- [ ] 增量收益不只依赖单一因子。

## 八、阶段 5：HMM 市场状态识别

### 1. 第一版输入变量

```text
benchmark_log_return
realized_volatility
market_breadth
log_market_amount
```

所有变量只能在训练窗口内标准化。

### 2. 实验组

- 无状态；
- 规则式状态；
- 两状态 Gaussian HMM；
- 三状态 Gaussian HMM；
- 三状态 Student-t HMM，作为后续稳健性实验。

### 3. 实时估计流程

每个调仓点：

1. 截取截至当日的扩展训练窗口；
2. 在训练数据上拟合 HMM；
3. 根据收益、波动率和宽度给状态排序；
4. 只计算 filtered probability；
5. 输出下一持有期可使用的状态概率；
6. 禁止将 smoothed probability 用作策略输入。

### 4. 状态概率映射

采用软概率而不是硬切换：

$$
a_t=1-\lambda_bP(\text{bear}_t)+\lambda_uP(\text{bull}_t)
$$

第一版限制：

```text
0.90 <= a_t <= 1.00
```

### 5. 稳定性检查

- [ ] 不同训练窗口的状态含义稳定。
- [ ] 状态没有异常高频跳变。
- [ ] 状态持续期具有经济合理性。
- [ ] 状态概率没有长期退化到 0 或 1。
- [ ] 不同初始化结果基本一致。
- [ ] 两状态和三状态模型具有可解释的对应关系。

### 6. 输出

```text
regime_probabilities.csv
regime_transition_matrix.csv
regime_duration.csv
regime_factor_performance.csv
regime_stability_audit.csv
```

### 晋级标准

- [ ] filtered probability 不存在未来依赖。
- [ ] 状态含义在多数滚动窗口中稳定。
- [ ] 回撤或尾部损失得到改善。
- [ ] 改善不只依赖单次危机。
- [ ] 状态调整没有造成成本和换手失控。

### 3. 技术路线门禁
