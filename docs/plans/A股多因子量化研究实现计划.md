# A股多因子量化研究作品高质量实现计划

> [!info] 项目定位
> 这个项目用于投递量化研究实习生岗位，重点展示：
> **数据清洗 → 因子构建 → 因子检验 → 组合回测 → 风险归因 → LLM辅助解释 → 研究报告输出**。

---

## 0. 项目一句话概括

**项目名称建议：**

> **《A股多因子选股策略研究：因子有效性检验、组合回测与LLM辅助解释》**

**项目目标：**

基于A股股票池，构建估值、动量、反转、波动率、流动性、规模、质量、成长、资金流与LLM事件标签等因子，使用Python完成数据清洗、因子检验、组合回测、风险分析和研究报告输出，形成一个可提交、可复现、可面试讲解的量化研究作品。

---

## 1. 为什么选择这个作品

### 1.1 与岗位要求的对应关系

| 岗位能力要求 | 项目中对应内容 |
|---|---|
| 数据与因子研究 | A股行情、财务、估值、资金流数据清洗，构建候选因子 |
| 策略回测与验证 | IC检验、Rank IC、分组回测、多因子组合回测 |
| Python数据分析 | pandas、numpy、matplotlib、statsmodels、sklearn |
| 统计基础 | 相关性检验、回归中性化、t检验、样本外检验 |
| AI辅助研究 | LLM新闻/公告事件标签提取、情绪分类、策略解释 |
| 研究报告输出 | README、Notebook、PDF报告、图表、代码结构 |

### 1.2 不建议做的作品

不建议做：

- 单只股票明天涨跌预测
- 简单K线技术指标买卖点
- 只展示收益曲线的回测
- 用复杂机器学习模型但无法解释
- 没有交易成本、没有基准比较、没有风险分析的策略

原因：

> 单只股票预测噪声大，容易变成普通机器学习作业；多因子截面研究更接近量化研究实习岗位的真实工作内容。

## 5. 项目最脆弱的前提

> [!warning] 面试官最可能追问的地方
> 高质量量化作品不是看收益曲线有多漂亮，而是看你有没有意识到回测中的关键偏差。

| 脆弱点 | 风险 | 处理方式 |
|---|---|---|
| 未来函数 | 财务数据、指数成分、复权价格可能引入未来信息 | 使用公告日期、调仓日滞后、下一交易日成交 |
| 生存者偏差 | 只用当前成分股会高估历史表现 | 尽量使用历史成分，无法获取时在报告中说明 |
| 交易不可达 | 涨停买不进、跌停卖不出 | 回测中加入涨跌停约束 |
| 成本低估 | 不考虑冲击成本会高估收益 | 设置多组成本压力测试 |
| 过拟合 | 因子太多、参数太多 | 样本内/样本外划分，做滚动检验 |
| LLM幻觉 | LLM生成错误标签 | LLM只做辅助解释，不直接作为核心买卖信号 |

## 9. 核心清洗规则

### 9.1 复权价格

构造复权收盘价：

```text
adj_close = close * adj_factor
```

收益率使用复权价格：

```text
return_t = adj_close_t / adj_close_{t-1} - 1
```

> [!note]
> 不要直接用未复权收盘价计算动量或收益率，否则分红送转会造成异常跳变。

---

### 9.2 财务数据 point-in-time 对齐

错误做法：

```text
2023年年报数据直接用于2023-12-31选股
```

正确做法：

```text
2023年年报如果公告日是2024-04-20，
则只能从2024-04-21或之后的下一个交易日开始使用。
```

实现逻辑：

```python
usable_date = next_trade_date(ann_date)
```

每个调仓日只使用：

```text
usable_date <= rebalance_date
```

的最新财务数据。

---

### 9.3 股票池过滤规则

每个调仓日执行：

```text
候选股票池 =
    指数历史成分股
    - ST股票
    - 停牌股票
    - 上市未满120个交易日股票
    - 退市股票
    - 当日无法交易股票
```

| 过滤项 | 规则 |
|---|---|
| 新股 | 上市未满120个交易日剔除 |
| ST | 调仓日处于ST状态剔除 |
| 停牌 | 调仓日停牌剔除 |
| 退市 | 退市后不再进入股票池 |
| 成交额过低 | 过去20日平均成交额过低的股票剔除 |
| 涨停 | 调仓买入时涨停则不能买入 |
| 跌停 | 调仓卖出时跌停则不能卖出 |

# 第二部分：因子构建

## 10. 候选因子总表

建议先做 **15-17个候选因子**，最后筛选出 **5-8个** 进入组合。

| 类别 | 因子名 | 计算方式 | 方向 |
|---|---|---|---|
| 动量 | MOM_20 | 过去20日收益率 | 正向或待检验 |
| 动量 | MOM_60 | 过去60日收益率 | 正向或待检验 |
| 反转 | REV_5 | -过去5日收益率 | 正向 |
| 波动率 | VOL_20 | 过去20日日收益率标准差 | 通常负向 |
| 波动率 | IDIO_VOL | 回归市场收益后的残差波动率 | 通常负向 |
| 流动性 | TURN_20 | 过去20日平均换手率 | 待检验 |
| 流动性 | AMIHUD | abs(return) / amount | 通常负向 |
| 规模 | SIZE | log(总市值) | 待检验 |
| 估值 | BP | 1 / PB | 通常正向 |
| 估值 | EP | 1 / PE_TTM | 通常正向 |
| 质量 | ROE | 净资产收益率 | 通常正向 |
| 质量 | GROSS_MARGIN | 毛利率 | 通常正向 |
| 杠杆 | LEVERAGE | 资产负债率 | 通常负向或待检验 |
| 成长 | REV_GROWTH | 营收同比增长率 | 通常正向 |
| 成长 | PROFIT_GROWTH | 净利润同比增长率 | 通常正向 |
| 资金流 | MF_20 | 过去20日主力净流入/成交额 | 通常正向 |
| LLM事件 | EVENT_SENTIMENT | 过去20日新闻公告情绪得分 | 待检验 |

---

## 11. 价格量价因子

### 11.1 MOM_20：20日动量

```text
MOM_20 = adj_close_t / adj_close_{t-20} - 1
```

### 11.2 MOM_60_SKIP5：跳跃动量

```text
MOM_60_SKIP5 = adj_close_{t-5} / adj_close_{t-60} - 1
```

作用：

> 避免最近几天短期反转干扰中期动量信号。

### 11.3 REV_5：短期反转

```text
REV_5 = -(adj_close_t / adj_close_{t-5} - 1)
```

含义：

- 过去5日涨得越多，反转因子越低
- 过去5日跌得越多，反转因子越高

### 11.4 VOL_20：20日波动率

```text
VOL_20 = std(daily_return, window=20)
```

### 11.5 AMIHUD：非流动性因子

```text
AMIHUD = mean(abs(return_t) / amount_t, window=20)
```

含义：

> 成交额越小、价格冲击越大，AMIHUD越高。

---

## 12. 基本面因子

> [!warning]
> 基本面因子必须使用公告日对齐，不能直接用报告期。

### 12.1 BP：账面市值比

```text
BP = 1 / PB
```

### 12.2 EP：盈利收益率

```text
EP = 1 / PE_TTM
```

### 12.3 ROE：盈利质量

```text
ROE = net_profit / net_assets
```

### 12.4 成长因子

```text
REV_GROWTH = revenue_t / revenue_{t-4} - 1
PROFIT_GROWTH = net_profit_t / net_profit_{t-4} - 1
```

# 第三部分：因子预处理

## 13. 缺失值处理

每个调仓日进行截面处理：

| 情况 | 处理 |
|---|---|
| 少量缺失 | 用行业中位数填充 |
| 行业内全缺失 | 用全市场中位数填充 |
| 缺失比例过高的因子 | 该期不使用该因子 |
| 财务因子缺失严重 | 限制在财报完整股票池中检验 |

---

## 14. 去极值：MAD方法

```text
median = median(x)
mad = median(|x - median|)
upper = median + 3 * 1.4826 * mad
lower = median - 3 * 1.4826 * mad
```

处理方式：

```text
x = min(max(x, lower), upper)
```

---

## 15. 标准化：截面 z-score

每个调仓日做截面标准化：

```text
z = (x - mean(x)) / std(x)
```

目的：

- 统一不同因子的量纲
- 方便因子合成
- 避免某个量纲过大的因子支配综合分数

---

## 16. 行业和市值中性化

对每个因子做横截面回归：

```text
factor_z = α + β1 * log_market_cap + 行业哑变量 + ε
```

取残差：

```text
factor_neutral = ε
```

意义：

> 剔除因子中由行业和市值带来的系统性暴露。

例子：

- 低PB因子可能天然偏向银行、地产
- 小市值因子可能天然偏向小盘成长股
- 如果不做中性化，策略收益可能只是行业或市值暴露

# 第四部分：因子有效性检验

## 17. 构造未来收益标签

默认使用未来20个交易日收益率：

```text
future_return_20 = adj_close_{t+20} / adj_close_t - 1
```

也可以使用相对基准收益：

```text
future_excess_return_20 = stock_return_20 - benchmark_return_20
```

主报告建议使用：

```text
未来20日截面收益率
```

附录补充：

```text
未来5日、10日、60日收益率
```

---

## 18. IC检验

每个调仓日计算：

```text
IC_t = corr(factor_t, future_return_t)
```

核心指标：

| 指标 | 公式 | 解释 |
|---|---|---|
| IC均值 | mean(IC_t) | 因子平均预测能力 |
| IC标准差 | std(IC_t) | 因子稳定性 |
| ICIR | mean(IC_t) / std(IC_t) | 单位波动下的预测能力 |
| IC胜率 | IC_t > 0 的比例 | 方向稳定性 |
| IC t值 | mean(IC_t) / se(IC_t) | 显著性 |
| Rank IC | Spearman相关 | 对异常值更稳健 |

---

## 19. 分组回测

每个调仓日按因子值分成5组：

```text
Q1 = 因子最低20%
Q2 = 20%-40%
Q3 = 40%-60%
Q4 = 60%-80%
Q5 = 因子最高20%
```

检验重点：

| 检验点 | 判断 |
|---|---|
| Q5是否长期高于Q1 | 因子方向是否有效 |
| 分组收益是否单调 | 因子是否稳定 |
| 多空组合Q5-Q1是否为正 | 因子是否有截面区分能力 |
| 不同年份是否稳定 | 是否只在某一年有效 |
| 不同市场环境是否稳定 | 牛市、熊市、震荡市是否失效 |

---

## 20. 因子相关性分析

计算因子之间的截面相关性：

```text
corr(factor_i, factor_j)
```

处理规则：

| 情况 | 处理 |
|---|---|
| 相关性 < 0.5 | 可以同时保留 |
| 相关性 0.5-0.8 | 谨慎保留，检查经济含义 |
| 相关性 > 0.8 | 通常只保留一个 |
| 同类因子高度相关 | 选择IC更稳定的那个 |

输出图：


# 第五部分：多因子合成与组合构建

## 21. 因子筛选规则

保留因子的标准：

| 标准 | 说明 |
|---|---|
| IC均值方向稳定 | 多数年份方向一致 |
| ICIR相对较高 | 预测能力更稳定 |
| 分组收益有单调性 | 高低组区分明显 |
| 因子相关性不过高 | 避免重复信号 |
| 有经济含义 | 不只依赖数据拟合 |
| 样本外仍有效 | 防止过拟合 |

---

## 22. 等权合成

基础版本：

```text
score = mean(selected_factor_z)
```

优点：

- 简单
- 稳健
- 容易解释
- 不容易过拟合

---

## 23. ICIR加权合成

进阶版本：

```text
weight_i = ICIR_i / sum(abs(ICIR_i))
score = Σ weight_i * factor_i
```

注意：

> 权重只能使用过去窗口计算，不能使用全样本ICIR。

例如：


## 25. 选股规则

默认规则：

```text
每月最后一个交易日计算因子；
下一个交易日开盘调仓；
选择综合得分最高的50只股票；
等权持有；
持有到下一个调仓日。
```

可测试方案：

| 方案 | 规则 |
|---|---|

| Top 50 | 默认方案，平衡收益与分散 |
| Top 100 | 更稳，但超额收益可能下降 |
| Top 10% | 股票池变化时更自然 |


# 第十三部分：面试准备

## 48. 面试高频问题

| 问题 | 回答方向 |
|---|---|
| 为什么选中证500？ | 成分股数量适中，流动性较好，比沪深300更有截面差异 |
| 为什么不用单只股票预测？ | 单只股票噪声大，多因子截面研究更符合量化选股逻辑 |
| 如何避免未来函数？ | 使用公告日、调仓日滞后、下一交易日成交 |
| 为什么做行业中性化？ | 防止因子收益来自行业暴露 |
| 为什么做市值中性化？ | 防止因子只是小市值或大市值暴露 |
| IC和Rank IC区别？ | IC是线性相关，Rank IC是排序相关，更适合截面选股 |
| 分组回测看什么？ | 看单调性、Q5-Q1收益、稳定性 |
| 为什么加入交易成本？ | 防止高换手策略被成本吞噬 |
| 策略什么时候失效？ | 市场风格切换、极端行情、因子拥挤、成本上升 |
| LLM在项目里做什么？ | 做文本摘要、事件标签、辅助解释，不直接替代统计检验 |
