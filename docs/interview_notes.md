# 项目面试说明

## 为什么做多因子

单一因子容易受市场阶段、行业结构和样本噪声影响。多因子框架可以把量价、基本面、资金流和事件信息放在统一的截面排序体系中，并通过 IC、分组收益、相关性、换手和回撤检查信号是否稳定。当前项目更强调研究流程和风控意识，不把样例收益当作真实结论。

## 为什么做中性化

中性化的目的是降低行业、市值等已知风险暴露对因子检验的干扰。例如小市值因子在某些阶段可能贡献大部分收益，如果不控制市值暴露，其他因子的 IC 可能只是小市值暴露的替代。项目中的中性化在每个信号日横截面内完成，只使用当日已知行业和市值字段，避免跨期信息泄漏。

## 如何避免未来函数

项目默认因子在信号日收盘后生成，组合在下一交易日生效。财务数据使用公告日和可用日控制 point-in-time，可用日前不进入因子计算。因子检验的目标收益是未来 20 个交易日收益，只用于评价，不回填到信号构建。真实版本还需要检查指数成分、ST 状态、停复牌、涨跌停和退市数据是否使用了未来信息。

## 回测局限

当前样例回测使用合成数据，虽然已经输出订单、成交和持仓，并考虑下一交易日开盘执行、手数约束、成本、停牌和涨跌停限制，但仍不能代表真实盘口深度和成交容量。样例指标用于验证系统可运行，不用于证明策略可交易。

## LLM 模块定位

LLM 模块用于事件标签结构化和报告解释，不默认作为核心买卖信号。真实使用时需要记录 prompt 版本、模型版本、文本发布时间、抓取时间、抽样复核结果和标签一致性。LLM 输出可能有幻觉或误分类，必须作为辅助信息处理。

## 三分钟讲解版本

这是一个 A 股多因子研究框架，按“数据清洗、因子构建、因子处理、因子检验、组合回测、风险归因、LLM 事件解释”组织。项目重点不是展示一个高收益回测，而是把量化研究中容易出错的时间对齐、成本、回撤、换手、行业暴露、交易可达性和数据偏差放进可复现流程。当前使用合成样例数据，真实数据接入后会优先补齐交易可达性、样本外验证和容量约束。
