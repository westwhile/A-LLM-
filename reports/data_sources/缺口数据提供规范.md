# 缺口数据提供规范（阶段 1D，2026-07-19）

7 张许可表的提供规范。每张表的字段、粒度、单位与硬性口径以对应 `*_review.md` 为准；
本文件是给你（或数据供应商导出人员）的合并速查。

## 通用规则

1. 文件格式：CSV（UTF-8）或 Parquet，文件名必须为 `<表名>.csv` / `<表名>.parquet`，放入同一目录后通知 Kimi 导入；
2. 表头必须与 `templates/<表名>.template.csv` 完全一致（可多列，不可少列、不可改名）；
3. 覆盖起点不晚于 2015-01-01，必须包含已退市、已剔除与 ST 股票，禁止用当前值回填历史；
4. 导出后记录：提供者/账号类型、导出人、导出日期、供应商版本或接口 ID、许可条款页；
5. 任何"供应商口径说明页"（字段 ID、单位、计算方式）请一并存档为脱敏摘要。

## 各表速查

| 表 | 主键 | 最关键硬口径 | 推荐来源 |
| --- | --- | --- | --- |
| index_member | index_code+ts_code+in_date | 区间不重叠；含剔除/退市；out_date 可空 | Wind/iFinD/Choice/Tushare Pro 历史成分 |
| daily_basic | trade_date+ts_code | 与 daily_bar 键缺失率 ≤20%；turnover_rate 注明分母口径 | Tushare Pro daily_basic + moneyflow |
| financial_indicator | ts_code+report_period+ann_date(+revision_id) | revision_date≥ann_date；usable_date 严格晚于两者且为开市日；保留全部修订 | Tushare Pro fina_indicator |
| industry | trade_date+ts_code | 注明分类体系与版本；生效日期真实 | Wind/iFinD 申万或中信历史 |
| suspension | ts_code+suspend_date | 区间不重叠；resume_date 可空 | Wind/iFinD 停复牌 |
| st_status | ts_code+start_date | 区间不重叠；end_date 可空 | Wind/iFinD ST 历史 |
| limit_price | trade_date+ts_code | 单位 CNY；与 daily_bar 键对齐 | Tushare Pro stk_limit |

## 提供后的流程

```powershell
# 1) 把文件放入例如 data/inbox/real-v1/
# 2) 通知 Kimi：我会先做 schema/主键/日期/覆盖率的导入前校验
# 3) 你在登记表签署页确认许可后，我把 license_status 改为 approved_for_research
# 4) 重新执行真实导入与六项审计，阻断项归零后冻结 data/standard/real-v1
```

在全部 11 张核心表通过前，真实门禁按设计继续保持 `blocked_by_missing_pit_tables`，不进入模型运行。
